Module 2 · Copilot Fundamentals

Context & prompt files

Copilot's output quality is bounded by the context it can see. This page covers how to supply context deliberately, and how prompt files let you package reusable, shareable prompts for repeatable tasks.

Sources of context

  • Cursor surroundings — code before and after the caret.
  • Open editors / tabs — neighboring files inform suggestions.
  • Explicit references — #file, #selection, #editor in chat.
  • Workspace participant — @workspace lets chat reason over the project.
  • Repository indexing / knowledge bases — broader, semantic context (plan-dependent).
ℹNote

Because the context window is finite, relevance beats volume. Attaching the two files that matter usually beats dumping ten files or a giant selection.

Prompt files

Prompt files (*.prompt.md) let you save a reusable prompt — instructions plus references — as a file in the repo, so a task can be run consistently by anyone on the team. They're ideal for standardised generation tasks ("scaffold a CRUD controller following our conventions").

---
mode: agent
description: Scaffold a .NET 8 CRUD controller for a given entity
---
Create an ASP.NET Core controller for the #file:../Models/Entity.cs entity.
- Use constructor injection for the repository.
- Return ActionResult<T> with proper status codes.
- Add XML doc comments and a matching xUnit test class.
Follow the patterns in #file:../Controllers/OrderController.cs.

Best practices

★Best Practice
  • Open or reference the files that define the types you're working with.
  • Use prompt files for tasks the team repeats, so results are consistent and reviewable.
  • Keep prompts and references version-controlled — they evolve with the codebase.
  • Remove noise (dead code, unrelated files) from the working context when generating.

Limitations

  • Context that isn't visible or referenced generally isn't used.
  • Very large references can crowd out the important parts within the token budget.
  • Prompt-file availability and syntax can vary by surface/version.
⚠Common Mistake

Expecting Copilot to follow a team convention it has never seen. If the convention isn't in the visible context, a custom-instructions file, or a referenced example, Copilot can't apply it.

✓Exam Tip

Distinguish prompt files (reusable, task-specific prompts you invoke) from custom instructions (always-on, repo-wide guidance). Both shape output, but they're used differently — the next page covers custom instructions.
