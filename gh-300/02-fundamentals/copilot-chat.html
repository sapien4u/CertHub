Module 2 · Copilot Fundamentals

Copilot Chat

Copilot Chat is a conversational assistant embedded in the IDE and on github.com. Use it to explain code, propose changes, generate tests, fix errors, and answer questions — with awareness of your current file, selection, or wider workspace.

How it works

Chat combines your natural-language message with contextual signals (the active file, your selection, referenced files, and — depending on surface — the workspace or repository). It responds conversationally and can propose code you apply into the editor.

Slash commands, participants, and variables

These shortcuts steer chat toward the right task and context (availability varies by IDE):

Type Example Purpose
Slash command /explain, /fix, /tests, /doc, /optimize Invoke a common task
Chat participant @workspace, @terminal, @vscode Scope context to a domain
Context variable #file, #selection, #editor Attach specific context 
@workspace /tests generate xUnit tests for OrderService.CalculateTotal,
covering empty orders, tax rounding, and negative quantities

Inline chat vs chat view

  • Inline chat works at the cursor for quick, in-place edits and questions.
  • Chat view/panel is better for longer, multi-turn conversations and exploration.

Best practices

★Best Practice
  • Be specific about the framework, version, and constraints ("Angular 18 standalone component with signals", ".NET 8 minimal API").
  • Attach the right context with #file/#selection or @workspace instead of pasting large blocks.
  • Iterate: refine the request based on the answer rather than starting over.
  • Ask Copilot to explain trade-offs, not just produce code.

Limitations

  • Chat can still hallucinate APIs or give outdated guidance — verify against docs.
  • Without the right context attached, answers are generic.
  • It doesn't execute your code or guarantee that proposed changes compile.

Enterprise considerations

Copilot Enterprise enhances chat on github.com with repository-aware answers, and organizations can enable knowledge bases (Enterprise) so chat can draw on curated documentation. Policies control chat availability and data handling.

⚠Common Mistake

Pasting huge code dumps into chat instead of referencing files. This wastes the context window and often produces worse answers than a focused #selection plus a precise question.

✓Exam Tip

Know the difference between slash commands (tasks like /fix, /tests), participants (@workspace), and context variables (#file). Exam questions often check whether you can match the tool to the goal.
