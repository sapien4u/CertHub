Module 2 · Copilot Fundamentals

Copilot in the CLI

Copilot in the CLI brings assistance to the terminal through the GitHub CLI (gh). It helps you find the right command, explain an unfamiliar one, and translate natural language into shell, git, or gh commands — without leaving the terminal.

How it works

After installing the gh CLI and the Copilot extension, you ask in plain language and Copilot proposes a command with an explanation. You review it, then choose to run, copy, or revise — the command is never executed without your confirmation.

# Install once
gh extension install github/gh-copilot

# Ask Copilot to suggest a command
gh copilot suggest "find files larger than 50MB and sort by size"

# Ask Copilot to explain a command
gh copilot explain "git rebase -i HEAD~3"

Typical uses

  • Recalling infrequent git operations (interactive rebase, reflog recovery).
  • Building gh commands for PRs, issues, and workflows.
  • Explaining a risky command before you run it.
  • Composing shell one-liners for file and text processing.

Best practices

★Best Practice
  • Always read the explanation and understand a command before running it — especially anything destructive (rm, git reset --hard, force-push).
  • Use explain to learn, not just suggest to act.
  • Prefer non-destructive alternatives when offered.

Limitations

  • Suggested commands can be wrong or unsafe for your specific environment.
  • Copilot doesn't know your machine's full state or side effects.
  • It won't run commands for you automatically — and that's by design.
⚠Common Mistake

Blindly pasting and running a suggested destructive command. The confirmation step exists precisely so you can catch mistakes; don't treat it as a formality.

✓Exam Tip

Know that CLI assistance uses gh copilot suggest and gh copilot explain, requires confirmation before running, and is delivered as a gh extension.
