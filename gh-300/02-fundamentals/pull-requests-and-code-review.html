Module 2 · Copilot Fundamentals

Pull requests & code review

Copilot extends into the collaboration stage of the lifecycle with pull-request summaries and AI code review, helping authors describe changes and reviewers catch issues faster.

PR summaries

On github.com, Copilot can generate a summary of the changes in a pull request — an overview and a walkthrough of modified files — which the author edits and keeps. This speeds up writing good PR descriptions and helps reviewers orient quickly.

ℹNote

The author remains responsible for the final description. Treat the generated summary as a first draft to refine, not a finished record.

Copilot code review

Copilot can review a pull request and leave suggestions — flagging potential bugs, style issues, and improvements, sometimes with proposed code changes you can apply. You can request a review from Copilot much like requesting one from a teammate.

  • Fast first pass that catches obvious issues before human reviewers spend time.
  • Suggestions are advisory — humans approve and merge.

Where each fits

Task Feature
Draft the PR description PR summary
Get an automated first-pass review Copilot code review
Approve and merge Human reviewer (always)

Best practices

★Best Practice
  • Use Copilot review as an early gate, not a replacement for human review.
  • Keep PRs small — Copilot (and humans) review focused changes better.
  • Verify each suggested change; apply only what genuinely improves the code.
  • Edit the generated summary to capture intent and risk, which the model can't fully infer.

Limitations

  • Copilot may miss domain-specific or architectural problems.
  • It can produce false positives or superficial comments.
  • It doesn't understand business intent or unwritten team standards unless provided.

Enterprise considerations

Availability of PR summaries and code review can depend on plan and repository settings; organizations may enable or restrict these features via policy.

⚠Common Mistake

Treating an AI review as sufficient sign-off. Human review and approval are still required — AI review augments reviewers, it doesn't replace accountability.

✓Exam Tip

Know that Copilot can summarise PRs and perform code review with suggestions, but final approval/merge is a human responsibility.
