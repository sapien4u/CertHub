Module 3 · Prompt Engineering

Iteration & prompt templates

Great results rarely come from the first prompt. Iteration — and capturing what works as templates — turns one-off wins into repeatable quality.

The refinement loop

  1. Draft a prompt with goal, context, and constraints.
  2. Generate and read the output critically.
  3. Diagnose the gap (missing edge case? wrong pattern? outdated API?).
  4. Refine the prompt or context and regenerate.
  5. Lock in the working prompt as a template or instruction.
Turn 1: "Add caching to GetProductAsync."
Turn 2: "Use IMemoryCache with a 5-minute sliding expiration keyed by id;
         invalidate on UpdateProductAsync; keep the method async."
Turn 3: "Add an xUnit test proving a cache hit avoids the repository call."

Each turn narrows toward the intended design without discarding progress.

Prompt templates

A template is a reusable prompt skeleton for a recurring task. Capture them where the team can reuse them — as prompt files (*.prompt.md) or in a shared prompt library.

# Template: new endpoint
Goal: Add a {{verb}} endpoint for {{resource}}.
Context: #file:{{controller}}, #file:{{model}}
Constraints: .NET 8 minimal API; validation via data annotations;
             return ProblemDetails on error; async with CancellationToken.
Format: updated endpoint + xUnit tests for validation failures.

Reusable building blocks

  • Constraint block — your standing rules (versions, patterns, security).
  • Format block — the exact output shape you want.
  • Context block — the standard files to reference for a task type.

Best practices

★Best Practice
  • Keep a personal/team prompt library for high-value tasks (endpoints, tests, migrations).
  • Version prompt files with the code so they evolve together.
  • Refine the context as readily as the wording — often it's the real bottleneck.
⚠Common Mistake

Restarting from scratch when an answer is close. Iterating on the existing thread preserves useful context and converges faster.

✓Exam Tip

Recognise prompt templates / prompt files as a way to standardise and share effective prompts across a team — a governance and consistency win, not just a convenience.
