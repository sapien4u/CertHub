Module 3 · Prompt Engineering

Prompting techniques

Beyond general principles, a handful of named techniques appear on the exam and are genuinely useful day to day.

Zero-shot prompting

You describe the task with no examples and let the model infer the pattern. Fast and often enough for common tasks.

Write a C# extension method that converts a string to a URL-friendly slug.

Few-shot prompting

You provide one or more examples of the desired input/output so the model matches your exact style. Use when output shape or conventions matter.

Map DTOs like these examples:
UserDto -> User: { Id, FullName -> Name, Email }
Now generate the mapper for OrderDto -> Order following the same style.
ℹNote

Zero-shot = no examples; one-shot = a single example; few-shot = a small number of examples. More/better examples generally improve adherence to your desired format.

Chain-of-thought / step-by-step

Ask the model to reason in steps or to plan before coding for complex tasks. Useful for multi-part logic and debugging.

Before writing code, list the steps to safely migrate this synchronous
repository to async, then implement step 1.

Role and persona framing

Setting a role can focus the response ("Act as a security reviewer and identify injection risks in #selection").

Decomposition

Break a big request into smaller prompts. Generate the interface, then the implementation, then the tests — reviewing each before moving on.

Technique selection

Situation Technique
Common, well-known task Zero-shot
Must match a specific style/format Few-shot
Complex reasoning or multi-step logic Step-by-step / decomposition
Focused perspective (security, perf) Role framing
★Best Practice

Combine techniques: a few-shot example plus an explicit constraint list plus decomposition yields the most controllable results on non-trivial tasks.

⚠Common Mistake

Using a single giant prompt for a large task and accepting the whole output. Decompose and review in stages to keep quality and reviewability high.

✓Exam Tip

Be ready to identify zero-shot vs few-shot from a description, and to pick the technique that best fits a given goal.
