Module 6 · Developer Productivity

.NET 8 & C#

Applied Copilot patterns for the .NET 8 / C# work that dominates your day: APIs, EF Core, DI, minimal APIs, tests, and refactoring.

Minimal APIs

// Prompt
Create a .NET 8 minimal API endpoint group for /products:
GET (paged), GET by id, POST (validated). Use records for DTOs,
constructor-injected IProductService, return TypedResults, and ProblemDetails on errors.

var products = app.MapGroup("/products");
products.MapGet("/", async (int page, int pageSize, IProductService svc, CancellationToken ct)
    => TypedResults.Ok(await svc.GetPageAsync(page, Math.Min(pageSize, 100), ct)));
products.MapGet("/{id:int}", async (int id, IProductService svc, CancellationToken ct)
    => await svc.GetAsync(id, ct) is { } p ? TypedResults.Ok(p) : TypedResults.NotFound());

EF Core

Copilot is strong at LINQ-to-EF, but you must watch for N+1 and tracking issues.

Rewrite this to a single EF Core query with projection and AsNoTracking;
it currently loads entities then queries per row.
★Best Practice

Ask Copilot to project to DTOs and use AsNoTracking() for read paths, and to explain the generated SQL. Verify with logging or a profiler — don't assume the query is optimal.

Dependency injection

Give Copilot your registration file as context so it wires services the way your app expects (scoped/singleton/transient) and follows your conventions.

Unit tests

/tests xUnit + FluentAssertions for ProductService.GetPageAsync:
clamps pageSize to 100, returns empty page past the end, throws on page < 1.

Refactoring to modern C#

Use edit mode to modernise: primary constructors, collection expressions, required members, pattern matching. Keep tests green across the change.

Common mistakes

⚠Common Mistake
  • Accepting EF queries that trigger N+1 or load whole tables into memory.
  • Letting Copilot hard-code connection strings/secrets — read from Key Vault/config.
  • Trusting generated tests that assert current behavior rather than intended behavior.
✓Exam Tip

For "developer productivity" questions, the theme is: Copilot accelerates idiomatic .NET tasks, but you supply context (types, conventions) and validate correctness, performance, and security.
