Module 5 · Enterprise Features

Security & access controls

This page consolidates the security and data-protection controls available to organizations using Copilot — the safeguards that make enterprise adoption defensible.

Data protection by plan

  • Business & Enterprise: prompts, suggestions, and your code are not used to train the foundation models; code snippets aren't retained for training. Data handling is designed for organizational protection.
  • Individual free tier: fewer guarantees; may allow using data to improve products unless opted out. Pro/Business/Enterprise provide stronger protections.
✓Exam Tip

A clean exam heuristic: if the scenario involves a company protecting its code and needing assurances it won't train public models, the answer is Business or Enterprise (not the free individual tier).

Access management

  • Seat/license assignment to users, teams, or organizations.
  • Integration with organization/enterprise membership and, where applicable, SSO/SAML and identity controls at the platform level.
  • Ability to revoke access by unassigning seats.

Content exclusions (recap)

Admins define files/paths/repos that Copilot must not use as context or send to the service — the primary control for keeping secrets and sensitive code out of prompts and suggestions.

Matching-public-code filter (recap)

Blocks suggestions that match public code; a prerequisite for IP indemnity on Business/Enterprise and a key control in regulated environments.

Auditing & monitoring

  • Audit logs for compliance and investigation.
  • Usage metrics / metrics API for oversight and adoption tracking.

Defence-in-depth checklist

★Best Practice
  • Enable content exclusions for secrets/config/sensitive domains.
  • Enable the matching-public-code filter (and gain IP indemnity).
  • Enforce least-privilege seat assignment; revoke on offboarding.
  • Monitor via audit logs and metrics.
  • Train developers to review, test, and scan all generated code.
  • Never rely on Copilot to keep secrets safe — keep them out of context entirely.
⚠Common Mistake

Treating Copilot's data protections as a substitute for your own controls. Exclusions, scanning, least-privilege, and review are still your responsibility; the plan protections complement them.

ℹNote

Security with Copilot is shared responsibility: GitHub provides plan-level protections and controls; the organization must configure and operate them and keep humans in the loop.
