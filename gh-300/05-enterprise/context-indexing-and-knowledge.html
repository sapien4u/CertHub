Module 5 · Enterprise Features

Context, indexing & knowledge bases

Enterprise plans extend Copilot's awareness beyond open files to the repository and to curated organizational knowledge. These features power more relevant, organization-specific answers.

Repository indexing & @workspace

Copilot can build a semantic index of a repository so chat (via @workspace / repository-aware chat) can retrieve relevant code across the project — not just what's open. This improves questions like "where is authentication handled?" or "how do we call the payments API?"

ℹNote

Indexing enables semantic retrieval across the codebase. It's what lets Copilot answer repo-wide questions rather than only reasoning over the current file.

Knowledge bases (Enterprise)

Copilot Enterprise lets organizations create knowledge bases from collections of Markdown documentation across repositories. Copilot Chat on github.com can then ground answers in that curated knowledge — internal standards, architecture docs, runbooks.

  • Keeps answers aligned with your documented practices.
  • Reduces reliance on generic public knowledge for internal questions.
  • Curated and maintained by the organization.
✓Exam Tip

Knowledge bases are an Enterprise feature built from Markdown docs, used to ground Copilot Chat in organizational knowledge. If a scenario needs Copilot to answer using internal documentation, the answer is Enterprise knowledge bases.

Layers of context (summary)

Layer Scope Availability
Cursor + open files Local All plans
#file / #selection references Explicit All (chat)
Repository indexing / @workspace Whole repo Broader on Enterprise-class surfaces
Knowledge bases Curated org docs Enterprise

Best practices

★Best Practice
  • Maintain high-quality Markdown docs — knowledge bases are only as good as their sources.
  • Use repo-aware chat for onboarding and cross-cutting questions.
  • Keep sensitive material governed by content exclusions even as you expand context.
⚠Common Mistake

Expecting knowledge-base-quality, organization-specific answers on non-Enterprise plans. Broad, curated org knowledge grounding is an Enterprise capability.
