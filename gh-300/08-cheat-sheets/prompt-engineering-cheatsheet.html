Module 7 · Cheat Sheets

Prompt engineering cheat sheet

A one-page reference. Print-friendly (use the print button).

The prompt formula

Goal + Context + Constraints + Format

  • Goal — what to produce.
  • Context — relevant types/files (reference, don't paste).
  • Constraints — versions, patterns, security, performance.
  • Format — the exact output shape.

Techniques

Technique Use when
Zero-shot Common, well-understood task
Few-shot Output must match a specific style/format
Step-by-step Complex logic or debugging
Decomposition Large task → smaller reviewed steps
Role framing Focused lens (security, performance)

Context levers

  • Open the defining files; keep a reference implementation visible.
  • Reference with #file, #selection, @workspace.
  • Cut noise; relevance beats volume.
  • Put standing rules in custom instructions; reusable prompts in prompt files.

Chat quick reference

Kind Examples
Slash commands /explain /fix /tests /doc /optimize
Participants @workspace @terminal @vscode
Variables #file #selection #editor

Good vs poor

Poor Better
"fix this" "guards null order.Lines, add a test"
"make it faster" "remove N+1; single EF query"
"write tests" "xUnit: empty, boundary, negative cases"
★Best Practice

Front-load the most important constraint, iterate rather than restart, and always read/verify the output.
