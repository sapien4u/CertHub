Module 4 · Responsible AI

Validation & human review

Every Responsible-AI thread converges here: generated code must be validated by a human before it ships. This page turns that principle into a concrete workflow.

Why validation is non-negotiable

  • Output is probabilistic — plausible isn't correct.
  • The model can't verify against your runtime, data, or requirements.
  • Accountability stays with the developer and the team, not the tool.

A validation workflow

  1. Read the suggestion fully — understand every line before accepting.
  2. Check intent — does it actually solve the requirement, including edge cases?
  3. Verify facts — do the APIs, packages, and parameters really exist?
  4. Test — run existing tests; add tests for new/edge behavior.
  5. Scan — SAST, dependency, and secret scanning.
  6. Review — human code review and approval before merge.
≡Example

Copilot suggests using a Enumerable.DistinctBy overload. Validation: confirm it exists for your target framework (.NET 6+), that it's the intended key, and add a test proving duplicates are removed by that key.

Understanding before accepting

If you can't explain what a suggestion does, you can't own it. "I don't fully understand this, but it works" is a red flag, not a shortcut.

★Best Practice

Make "explainability" a merge criterion: the author must be able to explain any Copilot-generated code as if they'd written it themselves. If they can't, it doesn't merge.

Where extra scrutiny is required

  • Security-sensitive code (auth, crypto, input handling).
  • Financial or safety-critical logic.
  • Data access and queries (correctness + injection).
  • Anything touching PII or secrets.

Best practices

★Best Practice
  • Apply the same gates to AI code as human code: review, tests, scans.
  • Add tests for the risky paths, since Copilot favors the happy path.
  • Escalate scrutiny with the stakes of the code.
⚠Common Mistake

Letting AI-generated code bypass normal quality gates because "it's just boilerplate" or "the AI is usually right." Uniform gates are what keep the failure rate low.

✓Exam Tip

If an answer choice emphasises human review, testing, and validation before shipping, it aligns with the core Responsible-AI message and is usually correct.
