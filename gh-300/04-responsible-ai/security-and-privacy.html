Module 4 · Responsible AI

Security & privacy

Copilot can accelerate secure development — and, if used carelessly, introduce risk. This page covers secure-coding responsibilities, prompt-injection awareness, and how Copilot handles your data.

Generated code is not pre-vetted

Suggestions can contain vulnerabilities learned from public code: injection flaws, weak crypto, missing authorization, hard-coded secrets. You own the security review.

// Risky: string-concatenated SQL Copilot might suggest
var sql = $"SELECT * FROM Users WHERE Name = '{name}'"; // SQL injection

// Secure: parameterised query
var user = await _db.Users.FirstOrDefaultAsync(u => u.Name == name);
★Best Practice

Keep the OWASP Top 10 in mind when reviewing generated code: injection, broken access control, cryptographic failures, SSRF, and friends. Route Copilot output through the same SAST, dependency, and secret scanning as any code.

Don't leak secrets or sensitive data

  • Never paste secrets, tokens, customer PII, or proprietary data into prompts unnecessarily.
  • Use content exclusions to keep sensitive files out of Copilot's context.
  • Read secrets from a vault (e.g., Azure Key Vault), never hard-code them — and don't let a suggestion talk you into it.

Prompt-injection awareness

Prompt injection is when untrusted content (a file, web page, issue, or dependency) contains instructions that try to manipulate the AI's behavior. As Copilot gains context from more sources (agents, web, repos), treat external content as data, not commands, and review actions an agent proposes.

⚠Common Mistake

Assuming text pulled into context is trustworthy. Malicious instructions embedded in a document or repo can attempt to steer generation. Review agent actions and generated code regardless of source.

How Copilot handles data (fundamentals)

  • Business and Enterprise: prompts and suggestions are not used to train the foundation models, and code snippets are not retained to train models. Business/Enterprise are built for organizational data protection.
  • Content exclusions let admins prevent specified files/paths from being used as context.
  • Duplication filter can block suggestions matching public code.
  • Telemetry and data-handling specifics differ by plan; Business/Enterprise offer stronger guarantees and admin controls than the individual free tier.
✓Exam Tip

Key privacy facts for the exam: with Copilot Business/Enterprise, your code/prompts are not used to train the models; content exclusions control what Copilot can see; and the matching-public-code filter controls what it can suggest.
